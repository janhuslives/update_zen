# update_zen

### *A tool to backup, update, and automatically rollback docker containers*

#### *I went from "fingers crossed" updates to boring, uneventful ones. That's all I wanted.*

</div>

---

> **update_zen is alpha-stage, non-production software made by a vibe coder.**
> This app can delete files and folders, change permissions, and modify running Docker containers. Sitting on your keyboard at the wrong moment can be disastrous. 
>**USE AT YOUR OWN RISK.**

---

## Overview ##

I made this app for myself because I needed to update some docker containers and I didn't want to irreversibly mess things up. I wanted to make sure I could *always* roll back to a working version of the app no matter what.
---

## What It Does

### 1 вҖ” Backs Up EVERYTHING

> Before the update, update_zen can archive the running container, attached volumes, yaml, env variables, and even the repository image. This lets you restore the system exactly as it was - no matter what happens next.
>
> **Full offline restore capability.**

---

### 2 вҖ” Has a User-Friendly Interface

> The app runs from the CLI but has an interactive, menu-driven interface. Data entry is kept to a minimum in favor of on-screen options.

| | Features |
|:---:|---|
| **a** | Manage Docker containers вҖ” start, stop, monitor logs, etc. |
| **b** | Pin versions without editing the YAML |
| **c** | Use custom repositories without touching the YAML |
| **d** | Encrypt anything you save to disk |
| **e** | Manage files вҖ” move and delete files and folders, change file permissions, etc. |
| **f** | Schedule and remove cron jobs for backups and updates |
| **g** | Backup or skip volumes and containers individually, based on rules and schedules |
| **h** | Build a customized CLI dashboard to monitor running Docker containers |

---

### 3 вҖ” Values Simplicity

> **One Python file runs the whole thing. No third-party libraries.**
>
> Portability is easy вҖ” the Python file is the app, the config file stores all your saved values. One optional credentials file may be created if you choose to save passwords. That's three files total, and for migrations, only the config file is truly important.

---

## Security

> This app is **small (~10k lines)** and completely open source. It works **offline.** It does not run in the background вҖ” it runs in the CLI, does its job, and stops. Total transparency at every level.

---

## Setup

### Prerequisites

| # | Requirement | Notes |
|:---:|---|---|
| 1 | **Docker** | `docker ps` should list your containers |
| 2 | **Python 3.x** | Check with `python3 --version` |
| 3 | **Sudo access** | Password prompted once, during install only |

> On Debian/Ubuntu, install Python 3 with: `sudo apt install python3`

---

### Installation

**Step 1 вҖ” Download the script**

```bash
wget https://raw.githubusercontent.com/janhuslives/update_zen/main/update_zen.py -O update_zen.py
```
> or copy directly to your server:
> scp update_zen.py user@server:~/update_zen.py


**Step 2 вҖ” Run the installer as your normal user**

```bash
python3 update_zen.py install
```

> You'll be prompted for your sudo password once. After that, no password is ever required again.

**Step 3 вҖ” Verify the install**

```bash
update_zen status
```

**Step 4 вҖ” Run it**

